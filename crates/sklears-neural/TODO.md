## OxiCUDA Migration (v0.2.0)

Status: fully migrated to oxicuda-* backends. Remaining work is hardening the
GPU path — making declared dependencies and capabilities real (Phase 5 of the
workspace migration plan; optional for the 0.2.0 release).

- [x] (L) Implement `tensor_core_conv2d` via oxicuda-dnn (declared dep currently unused) — done 2026-07-06: `GpuContext::tensor_core_conv2d` now builds a `DnnHandle` from the bound `Context`, NCHW `TensorDesc`/`TensorDescMut` descriptors + a `ConvolutionDescriptor::conv2d`, and calls `oxicuda_dnn::conv::api::conv_forward`. On `DnnError::WorkspaceRequired(bytes)` it allocates exactly that many bytes and retries once. `oxicuda-dnn` is now a real, referenced dependency. Files: `src/gpu.rs`.
- [x] (M) Wire real tensor-core / compute-capability detection — done 2026-07-06: `compute_capability` queries `oxicuda_driver::Device::compute_capability` (COMPUTE_CAPABILITY_MAJOR/MINOR) via `self.inner.context().device()` (no duplicate driver init — reuses the `Context` already owned by `sklears_core::gpu::GpuBackend`); `has_tensor_cores` derives `major >= 7` from it. This makes `tensor_core_gemm_f16`/`mixed_precision_gemm`/`tensor_core_matrix_multiply`'s f16 path and `tensor_core_recommendations` runtime-real (they still honestly report "not available" on this no-GPU host / on GPUs older than Volta). Files: `src/gpu.rs`.
- [x] (S) Resolve unused oxicuda-ptx and oxicuda-driver direct deps in the `gpu` feature — done 2026-07-06: `oxicuda-driver` is now genuinely used (`compute_capability` above, via `oxicuda_driver::Device`). `oxicuda-ptx` was dropped from `Cargo.toml` and the `gpu` feature — this crate's activation needs (relu/sigmoid/tanh) are already covered by `oxicuda_blas::elementwise`, and the conv2d path resolves algorithms/PTX internally inside `oxicuda-dnn`'s own `DnnHandle`, so no direct oxicuda-ptx usage was needed here. Files: `Cargo.toml`, `src/gpu.rs`.
- [x] (M) Honor `GpuConfig` `memory_pool_size`/`max_streams`/`mixed_precision`; implement `memory_pool_stats` — done 2026-07-07: new `src/gpu_pool.rs` submodule (`mod gpu_pool;` gated `#[cfg(feature = "gpu")]` in `lib.rs`) adds `PoolTelemetry` (pure atomic-counter `hits`/`misses`/`allocated_bytes`/`capacity_bytes`, hardware-independent, unit-tested without CUDA) and `GpuMemoryPool` (wraps a real `oxicuda_memory::MemoryPool`, gated on the newly-enabled `oxicuda-memory/pool` Cargo feature). `GpuContext` now stores real `memory_pool: GpuMemoryPool` and `streams: Vec<Stream>` fields built in `with_config` from `config.memory_pool_size`/`config.max_streams` (both genuinely read; the `#[allow(dead_code)]` on `config` is gone). Added `GpuContext::stream(index)` (round-robin) and `use_mixed_precision()` (`config.mixed_precision && has_tensor_cores()`, surfaced in `performance_stats` as `mixed_precision_active`; `mixed_precision_gemm`'s internals untouched, see next line). `memory_pool_stats()` now returns `self.memory_pool.telemetry_stats()` — real `(used_fraction, hit_rate)`; `performance_stats()`'s `HashMap` keys renamed `f32_pool_hit_rate`/`f64_pool_hit_rate` → `pool_used_fraction`/`pool_hit_rate` to match (no other crate read the old keys — each crate's own `GpuConfig`/`performance_stats` is unrelated). Deviation from the original plan: `GpuMemoryPool::acquire` returns a `PooledHandle<T>` wrapper rather than a bare `oxicuda_memory::PooledBuffer<T>`, because `PooledBuffer::drop` unconditionally frees back to the *underlying* pool with no caller hook — `PooledHandle::drop` instead returns the buffer to `GpuMemoryPool`'s own free-list, so a hit is a proven zero-allocation reuse rather than a guess about the underlying pool's opaque internal recycling. `GpuContext::with_config` performs one real single-element warm-up acquire/release so `GpuMemoryPool`/`PooledHandle`/`stream()` all have a genuine non-test production caller (fail-fast on a broken pool at construction time, not silently on first workload). Tests: 9 new (`PoolTelemetry` hit-rate/used-fraction/free-accounting/no-NaN cases, all hardware-independent; one live `GpuMemoryPool::acquire` hit/miss test that gracefully skips without a real CUDA device). `test_gpu_config_default` unchanged/still passing. `cargo build`/`clippy -D warnings` clean both with and without `--all-features`. Files: `src/gpu.rs`, `src/gpu_pool.rs` (new), `src/lib.rs`, `Cargo.toml`.
