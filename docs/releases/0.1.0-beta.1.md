### Recommended Configurations

#### Small Datasets (<10K samples)
```rust
// Use serial implementations for lower overhead
let model = LinearRegression::new()
    .parallel(false)  // Disable parallelism
    .fit(&x_train, &y_train)?;
```

#### Large Datasets (>100K samples)
```rust
// Enable parallelism for better throughput
let model = LinearRegression::new()
    .parallel(true)   // Enable parallel processing
    .n_jobs(4)        // Use 4 cores
    .fit(&x_train, &y_train)?;
```

#### High-Dimensional Data (>512d)
```rust
// SIMD gains diminish above 512 dimensions
// Focus on memory layout optimization
let model = KMeans::new(n_clusters)
    .algorithm(Algorithm::Lloyd)  // Cache-friendly
    .fit(&x_train)?;
```

## ðŸ› **Known Issues**

### Intermittent Test Failure
- **Test**: `sklears-impute::tests::property_tests::accuracy_tests::test_gaussian_process_imputer_accuracy`
- **Status**: Fails in parallel suite, passes when run individually
- **Cause**: Race condition in parallel test execution
- **Impact**: None (test is stable, just timing-sensitive)
- **Workaround**: Run package tests in isolation for consistent results

### SIMD Cache Effects
- **Issue**: SIMD speedup diminishes above 512 dimensions
- **Cause**: L2/L3 cache pressure on large vectors
- **Impact**: Performance plateaus at ~1Âµs for 1K+ dimensions
- **Status**: Expected behavior, not a bug
- **Recommendation**: Use blocking strategies for very high-dimensional data
