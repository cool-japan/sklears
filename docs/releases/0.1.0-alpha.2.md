# sklears 0.1.0-alpha.2 Release Notes

**Release date:** December 22, 2025  \
**Commit baseline:** `main` @ 0.1.0-alpha.2  \
**Workspace tests:** 11,292 passed · 170 skipped · 0 failed

## Overview

The second alpha release for sklears continues to build on the strong foundation established in alpha.1, with significant improvements in test coverage, stability, and documentation. This release adds over 1,200 new tests and refines the API surface ahead of the upcoming beta milestone.

## Highlights

- **Expanded Test Coverage** – Test count increased from 10,013 to 11,292 (+12.8%), strengthening confidence in API correctness and edge case handling across all modules.
- **Continued API Refinement** – Incremental polish to public interfaces based on early adopter feedback from alpha.1.
- **Documentation Improvements** – Updated README files and module documentation across all 35+ crates.
- **Dependency Updates** – Aligned with latest stable versions of core dependencies for improved compatibility.

## Changes Since Alpha.1

### Test Suite
- Added 1,279 new tests across the workspace

- Enhanced integration test scenarios

### Documentation
- Updated all crate README files with current version references

### Stability
- Fixed edge cases discovered through expanded testing

## Compatibility Notes

- Rust 1.70+ remains the minimum supported version.
- SciRS2 dependency stack compatibility maintained.
- No breaking API changes from alpha.1 — upgrade should be seamless.

## Upgrade Guide

1. Update your `Cargo.toml`:
   ```toml
   [dependencies]
   sklears = "0.1.0-alpha.2"
   ```
2. Run `cargo update` to pull the latest dependencies.
3. No code changes required for projects migrating from alpha.1.
